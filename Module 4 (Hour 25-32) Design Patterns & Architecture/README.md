# Module 4 - Design Patterns & Architecture

This module covers `Hour 25` through `Hour 32` with a consistent hour-based structure.

## Folders

- `Hour 25`: Clean Architecture theory
- `Hour 26`: Folder structure implementation
- `Hour 27`: Repository pattern
- `Hour 28`: Dependency Injection
- `Hour 29`: Factory and Strategy pattern
- `Hour 30`: Adapter and Observer pattern
- `Hour 31`: Event-driven internal architecture
- `Hour 32`: Mock test on refactoring an API to clean architecture

## Learning flow

Start with architecture concepts, then move into project structure, abstractions, composition, reusable patterns, and finally a refactoring exercise that combines the module topics.

## Suggested usage

Open each hour folder and use:

- `PROMPT.md` for the coding exercise prompt
- `README.md` for the topic explanation, learning goals, and expected implementation
